# LifeSync AI Webpage Analysis (Static Code Analysis)

본 문서는 로컬 웹 서버(`npm run dev`)를 직접 실행할 수 없는 환경에서, 프로젝트의 주요 뷰(`views/`) 컴포넌트 소스코드를 기반으로 UI 구조와 사용자 경험(UX) 기능을 역산하여 분석한 결과입니다.

## 1. 전역 아키텍처 및 레이아웃 (`App.tsx`)
- **싱글 페이지 애플리케이션(SPA)**: `App.tsx`가 라우터 및 글로벌 상태 관리자 역할을 수행합니다.
- **주요 탐색 기능(Nav/Sidebar)**: 화면 스크롤, 탭 전환을 통해 5개 이상의 메인 뷰(Journal, Todo, Calendar, Chat, CommunityBoard, Settings)를 전환합니다.
- **다크/라이트 테마**: 코드 내부적으로 색상 코드가 고정되어 있거나 테마별 유틸리티가 조합되어 사용됩니다 (예: `#37352f` 등 Notion 스타일의 색상 팔레트 채용).
- **데이터 저장 모델**: 로컬 스토리지(`loadFromStorage`, `saveToStorage`) 및 API 기반 클라우드(Supabase) 싱크를 병행합니다.

---

## 2. 개별 화면 (Views) 상세 분석

### 2.1. 일기장 / 메모뷰 (`JournalView.tsx`)
**[UX 컨셉: Notion 스타일의 미니멀리즘 메모장]**
- **레이아웃**: 좌측(또는 상단)에 메모 리스트, 우측(또는 하단)에 에디터 및 뷰어가 배치되는 구조.
- **상세 기능**:
  - `isListOpen` 토글을 통한 상단에 목록 축소/확장(5줄/10줄/20줄 보기).
  - 글쓰기 모드(`isWriting`) 시 포커스 뷰가 활성화되며 제목, 본문, 카테고리를 직관적으로 작성 가능.
  - **✨ AI 반응 시스템**: 각 메모마다 하단에 `AI의 반응(comments)` 영역이 렌더링 됩니다. 일기를 남기면 등록된 AI 페르소나들이 댓글을 달아 다이어리에 공감해주는 소셜 인터랙션 코드가 포함되어 있습니다.

### 2.2. 할 일 관리 뷰 (`TodoView.tsx`)
**[UX 컨셉: Trello + 무한 캔버스 칸반 보드]**
- **레이아웃**: 가로 방향으로 스크롤이 가능한 무한 캔버스(`isPanning`) 시스템. 마우스 드래그로 캔버스 자체를 이동할 수 있습니다.
- **상세 기능**:
  - `@dnd-kit` 라이브러리를 활용한 드래그 앤 드롭 지원. (리스트 통째로 순서 변경, 투두 아이템을 이리스트 저리스트로 이동)
  - 할 일은 완료됨(`showCompleted` 토글)과 미완료로 나뉘어 시각적 클러터를 줄입니다.
  - 인라인 수정(텍스트를 클릭하면 즉시 `<input>`으로 전환)을 지원해 빠른 작업 흐름을 제공합니다.

### 2.3. 캘린더 뷰 (`CalendarView.tsx`)
**[UX 컨셉: 깔끔한 Grid 기반 월간 캘린더]**
- **레이아웃**: 7x5 또는 7x6 그리드의 정통적인 월 단위 달력 폼을 지닙니다.
- **상세 기능**:
  - **Double Click to Add**: 날짜를 더블 클릭하면 해당일의 새 일정 모달창 표시.
  - **마우스 휠 스크롤 지원**: 월을 휠(Scroll)로 가볍게 넘길 수 있습니다(`handleWheel`).
  - **모던 모달(`showModal`)**: 모바일 스타일의 시간 피커(Hour/Min 별도 모달 토글)와 태그 뱃지를 지원합니다.

### 2.4. AI 채팅 뷰 (`ChatView.tsx`)
**[UX 컨셉: 컨텍스트 인지형 페르소나 챗봇]**
- **레이아웃**: 모바일 친화적인 말풍선(Bubble) 기반 채팅 레이아웃.
- **상세 기능**:
  - **온보딩 프로세스 (`onboardingStep`)**: 처음 접속 시 이름(`userName`)을 물어보며 환영 인사를 건넵니다.
  - **다중 에이전트 및 챗 모드 (`chatMode`)**: 기본, 롤플레잉, 학습 등 여러 모드 및 페르소나가 사용됩니다.
  - **의도 파악 (Intent Detection)**: `@google/genai`를 활용하여 유저가 텍스트로 보낸 내용으로부터 캘린더 추가(`add_event`), 할일 추가(`add_todo`), 일기 추가(`add_journal`) 액션을 역추출하여 실제 UI(데이터)에 반영합니다.
  - 리치 텍스트 렌더링 (`renderRichMessage`): 챗봇의 마크다운 응답(볼드, 리스트)을 해석하여 가독성 높게 표시합니다.

### 2.5. 커뮤니티 / AI 피드 보드 (`CommunityBoardView.tsx`)
**[UX 컨셉: 페르소나별 피드 뷰어]**
- **레이아웃**: AI 페르소나가 작성한 생각(포스트)을 구경할 수 있는 위에서 아래로 흐르는 단방향 피드 뷰.
- **상세 기능**:
  - `posts` 중에서 현재 선택된 페르소나(`selectedAgentId`)의 글을 필터링.
  - 유저(나)가 AI의 포스트에 역으로 댓글(`onAddComment`)을 다는 등 반전된 주객 반응 창구를 제공합니다.

## 3. 총평 및 개선 제안 (Risks/Notes)
1. **의존성 집중도**: `App.tsx` 파일 크기가 2,700줄에 달합니다. 데이터 로직(Supabase Sync 등)을 별도의 커스텀 훅(e.g., `useDataSync`)으로 완전히 분리하면 더 효율적인 코드베이스가 됩니다.
2. **모바일 최적화**: 코드 내에 모바일 터치 이벤트보다 마우스 클릭 베이스의 이벤트(DnD, 무한 스크롤 패닝 등)가 얽혀있습니다.
3. **디자인 통일성**: #37352f, #f7f7f5 등 차분하고 미니멀한(Notion-like) 컬러 시스템이 전반적으로 통일성 있게 사용되어 있어 매우 깔끔한 UX를 제공할 것으로 보입니다.
