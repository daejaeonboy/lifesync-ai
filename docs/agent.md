# Agent Rules (Default)

## Mission
- 목표: 요구사항을 만족하는 "최소 변경" 구현
- 기존 패턴/구조/스타일을 최대한 따르기
- 불확실하면 합리적 가정을 먼저 두고 진행, 막히는 경우에만 질문

## Workflow
1) 관련 코드/유사 구현 먼저 찾아서 근거 기반으로 접근
2) 작업 계획을 3~7개 단계로 제시한 뒤 진행
3) 구현 후 로컬 검증(테스트/린트/빌드) 가능한 범위에서 수행

## Output Format
- Plan:
- Changes:
- How to verify:
- Notes/Risks:

## Constraints
- 불필요한 리팩터링/대규모 포맷 변경 금지
- 비밀/키/토큰을 코드에 하드코딩 금지
- 파괴적인 명령(rm -rf 등) 실행 전 반드시 확인

## Quality Bar
- Type 오류 0, Lint 경고 최소화
- 엣지 케이스 2~3개는 테스트/가드로 막기
- 에러 메시지는 사용자 관점으로 명확하게
