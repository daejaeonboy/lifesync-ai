# 비즈니스 모델 및 성장 전략

> LifeSync AI의 수익화 및 서비스 확장을 위한 단계별 전략

---

## 🏗️ 전략 개요
**"사용자 주도(BYOK)로 시작하여 편의성 중심의 구독 모델(SaaS)로 전환"**

초기에는 개발 리소스와 운영 비용을 최소화하며 핵심 가치(AI Life Assistant)를 검증하고,
사용자 기반이 확보되면 고도화된 기능과 편의성을 앞세워 수익화를 진행합니다.

---

## 🚀 단계별 실행 계획

### Phase 1: MVP (Minimum Viable Product) - 현재 단계
**방식: BYOK (Bring Your Own Key)**

사용자가 직접 자신의 API 키를 발급받아 사용하는 구조입니다.

*   **구조**:
    *   사용자가 Google AI Studio에서 Gemini API Key를 발급받음 (무료 티어 활용 가능).
    *   앱 내 `설정` 페이지에 API Key 입력.
    *   Key는 서버에 전송되지 않고 사용자의 브라우저(`localStorage`)에만 안전하게 저장됨.
    *   AI 요청 시 클라이언트(브라우저)에서 Google로 직접 통신.
*   **장점**:
    *   **운영 비용 "0원"**: 서버 비용 및 토큰 비용이 발생하지 않음.
    *   **보안 리스크 최소화**: 개발자의 키가 유출될 일이 없음.
    *   **빠른 검증**: 서버 구축 없이 즉시 배포 가능.
*   **단점**:
    *   사용자 경험(UX) 장벽 존재 (키 발급 과정의 번거로움).

### Phase 2: 상용화 (Commercialization)
**방식: SaaS (Subscription Model)**

월 구독료를 지불하고 바로 서비스를 이용하는 구조입니다.

*   **구조**:
    *   **Proxy Server (Backend)** 도입 (Vercel Functions / Supabase Edge Functions 등).
    *   사용자의 요청 → Proxy Server (개발자의 API Key 주입) → Google AI.
    *   API Key는 서버 환경 변수로 철저히 은닉 (Client 노출 X).
*   **비즈니스 모델**:
    *   **Freemium**: 기본 기능 무료, AI 고급 분석은 월 구독 모델 (예: 월 4,900원).
    *   **Usage Cap**: 과도한 사용 방지를 위한 쿼리 제한(Rate Limiting) 적용.
*   **장점**:
    *   **최상의 UX**: 복잡한 설정 없이 가입 즉시 사용 가능.
    *   **수익 창출**: 안정적인 현금 흐름 확보.
*   **전환 시점**:
    *   Active User 1,000명 돌파 시점 또는 사용자들의 "키 발급 어렵다"는 피드백이 임계치를 넘을 때.

---

## ⚠️ 핵심 원칙 (Security First)
1.  **NO Client-side Keys**: 개발자의 API Key는 절대 클라이언트 코드에 포함하지 않는다.
2.  **Local Storage 암호화**: (Phase 1) 사용자가 입력한 키는 가능하면 암호화하여 로컬에 저장한다.
3.  **투명성**: 데이터가 어디로 전송되는지 명확히 고지한다 (Phase 1: Google로 직접 전송 / Phase 2: 우리 서버를 경유).
